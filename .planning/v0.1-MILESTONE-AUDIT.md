## Integration Check

**Integration checker result:** 16/17 requirements wired correctly. 3/3 E2E flows complete. 109/109 tests pass.

### Integration Findings

#### FIX-03 — Requirement Wording Mismatch (non-blocking)

- **Requirement:** `infer_freq emits UserWarning when a 1-day timedelta is detected, noting it cannot distinguish 'B' from 'D'`
- **Implementation:** `core.py` lines 126–129 resolve B-vs-D silently via weekday detection (`has_weekend` check). No `warnings.warn` call. The docstring at line 104 describes the limitation, but the function actually resolves it.
- **Tests:** `test_infer_freq_daily_weekdays_returns_B` and `test_infer_freq_daily_with_weekends_returns_D` verify resolution behavior — not warning emission.
- **Assessment:** The implementation is functionally superior to the requirement (smart resolution beats a confusing warning). Requirement text should be updated to describe actual behavior.

#### Dead `import numpy as np` in `_detect_plotly_tickformat` (latent risk)

- **Location:** `src/pxts/plots.py` line 174 (inside function body)
- **Issue:** numpy is imported but never called — `pd.Series.median()` is used instead. Leftover from Phase 3 plan description that specified numpy median.
- **Risk:** Latent `ImportError` if numpy is absent. numpy is not listed in `pyproject.toml` dependencies.
- **Fix:** Remove `import numpy as np` from inside `_detect_plotly_tickformat`.

#### `tmp_csv` Fixture Unused (minor)

- **Location:** `tests/conftest.py` lines 22–27
- **Issue:** Fixture defined but not referenced by any test function. Test IO functions create their own inline CSVs via `tmp_path`.
- **Impact:** Dead infrastructure — no coverage gap.
