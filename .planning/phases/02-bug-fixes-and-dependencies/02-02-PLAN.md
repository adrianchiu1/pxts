elif _tz_equal(index.tz, tz):

Add a module-level helper `_tz_equal` before `set_tz`:

```python
def _tz_equal(tz_a, tz_b) -> bool:
    """Return True if tz_a and tz_b represent the same timezone.

    Compares canonical keys via pd.Timestamp to handle aliases like
    'UTC' vs 'Etc/UTC' vs pytz.UTC.
    """
    try:
        key_a = str(pd.Timestamp("2000-01-01", tz=tz_a).tz)
        key_b = str(pd.Timestamp("2000-01-01", tz=tz_b).tz)
        return key_a == key_b
    except Exception:
        return False
```

Using a fixed timestamp (`2000-01-01`) avoids DST ambiguity during comparison. The `except` guard handles any exotic tz string that pd.Timestamp cannot parse — in that case we fall through to the conversion branch, which is the safe default.

The existing `test_set_tz_already_utc_returns_same_object` test covers the UTC case. Add a new test in test_core.py for `set_tz(df_utc, 'Etc/UTC')` asserting no warning and same object returned.
