**Mocking strategy for backends:**

Both `tsplot` and `tsplot_dual` accept an explicit `backend=` parameter, which bypasses
`get_backend()` entirely. Use `backend="matplotlib"` or `backend="plotly"` in tests.

For matplotlib: use `matplotlib.use("Agg")` (non-interactive) before importing pyplot.
For plotly: no display needed — plotly figures are data objects, no renderer called.

The `plt.show()` function is not called by pxts — `tight_layout()` and figure creation
are the only side effects. Use `matplotlib.use("Agg")` at module level in the test file.
